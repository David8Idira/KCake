## 1. 产品简介

KCake 是一个**异构设备集群化超大规模模型推理引擎**，可以将手机、电脑、服务器等不同设备组成集群，协同运行超出单一设备能力范围的大语言模型。

### 1.1 主要特性

- 🌐 **跨平台**：支持 iOS、Android、macOS、Windows、Linux
- 🔧 **异构计算**：CPU、GPU、NPU、TPU 协同工作
- 📦 **集群化**：多设备协同，突破单设备算力限制
- 🔌 **API 兼容**：OpenAI 和 Ollama 双重兼容
- 💻 **简单易用**：一条命令即可启动服务

## 2. 快速入门

### 2.1 安装

```bash
# 克隆项目
git clone https://github.com/liq_idira/KCake.git
cd KCake

# 安装依赖
pip install -r requirements.txt

# 安装 KCake
pip install -e .
```

### 2.2 启动服务

```bash
# 启动 API 服务器（单机模式）
python -m kcake serve --model meta-llama/Llama-3.1-8B

# 或指定集群模式
python -m kcake serve \
  --model meta-llama/Llama-3.1-8B \
  --cluster-key mysecret
```

### 2.3 访问服务

服务启动后，访问 http://localhost:8000 查看 Web UI。

```bash
curl http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{

    "messages": [
      {"role": "system", "content": "你是一个有帮助的助手"},
      {"role": "user", "content": "Hello!"}
    ],
    "temperature": 0.7,
    "max_tokens": 1000
  }'
```

#### Ollama 兼容格式

```bash
curl http://localhost:8000/api/generate \
  -H "Content-Type: application/json" \
  -d '{
    "model": "llama3.1",
    "prompt": "Explain quantum computing in simple terms",
    "stream": false
  }'
```

## 4. 集群使用

### 4.1 加入集群

```bash
# 在工作节点上执行
python -m kcake worker \
  --cluster-key mysecret \
  --name my-laptop
```

### 4.2 查看集群状态

```bash
curl http://localhost:8000/cluster/status
```

### 4.3 节点命名规范

| 节点类型 | 命名示例 |
|----------|----------|
| 服务器 | `server-gpu-01`, `server-cpu-02` |
| 桌面电脑 | `macbook-pro`, `windows-pc` |
| 手机 | `iphone-15`, `pixel-8` |
| 平板 | `ipad-pro`, `galaxy-tab` |

## 5. 参数配置

### 5.1 模型参数

| 参数 | 说明 | 默认值 | 建议范围 |
|------|------|--------|----------|
| `temperature` | 创造性温度 | 0.7 | 0.1-1.0 |
| `top_p` | 核采样 | 0.9 | 0.1-1.0 |
| `max_tokens` | 最大生成 Token | 2048 | 1-8192 |
| `stop` | 停止符 | - | 自定义 |

### 5.2 量化参数

| 量化类型 | 内存占用 | 性能 | 适用场景 |
|----------|----------|------|----------|
| FP16 | 100% | 最高 | 高精度场景 |
| INT8 | 50% | 高 | 平衡场景 |
| INT4 | 25% | 中 | 低显存设备 |

```bash
# 启动时指定量化
python -m kcake serve --model meta-llama/Llama-3.1-8B --quantization int4
```

## 6. 常见问题

### Q1: 首次使用需要下载模型吗？

是的，首次使用会自动从 HuggingFace 下载模型。也可以手动下载：

```bash
# 安装 huggingface-cli
pip install huggingface-hub

# 下载模型
huggingface-cli download meta-llama/Llama-3.1-8B
```

### Q2: 支持哪些模型？

支持所有 HuggingFace 上的主流模型，包括：
- LLaMA 系列
- Qwen 系列
- DeepSeek 系列
- GLM 系列
- Kimi 系列
- MoE 模型 (Mixtral, DeepSeek-V2 等)

### Q3: 手机可以作为工作节点吗？

可以。iOS 和 Android 设备可以作为工作节点加入集群，提供算力支持。

### Q4: 如何确认集群正常运行？

```bash
# 检查健康状态
curl http://localhost:8000/health

# 检查节点数量
curl http://localhost:8000/cluster/status | jq .total_nodes
```

## 7. 客户端示例

### 7.1 Python

```python
import openai

openai.api_base = "http://localhost:8000/v1"

response = openai.ChatCompletion.create(
    model="llama3.1",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```

### 7.2 JavaScript/Node.js

```javascript

  headers: { 'Content-Type': 'application/json' },

    model: 'llama3.1',
    messages: [{ role: 'user', content: 'Hello!' }]
  })
});

const data = await response.json();
console.log(data.choices[0].message.content);
```
