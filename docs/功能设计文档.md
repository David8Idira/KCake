## 2. 核心模块设计

### 2.1 kcake.core - 推理引擎

**职责**：
- 模型加载与分片
- Token 生成管理
- KV Cache 管理

**核心类**：

```python
class InferenceEngine:
    async def load_model()           # 加载模型
    async def generate(request)      # 生成文本
    async def generate_batch(reqs)  # 批量生成
    def get_model_info()            # 获取模型信息
    async def unload_model()        # 卸载模型
```

**数据流**：
```
User Input → Tokenize → Model Forward → Decode → Output
                         ↑
                   KV Cache ← Attention
```

### 2.2 kcake.heterogeneous - 异构调度器

**职责**：
- MoE 专家分类（Hot/Warm/Cold）
- NUMA-aware 内存分配
- 动态专家迁移

**核心类**：

```python
class HeteroScheduler:
    def register_device(device_id, type, memory)    # 注册设备
    def register_moe_experts(expert_ids, params)    # 注册专家
    def update_expert_frequency(expert_id, freq)    # 更新频率
    async def schedule_experts(expert_ids, devices)  # 调度专家
    def get_numa_optimal_mapping(expert_ids)         # NUMA优化
```

**专家分类策略**：

| 类型 | 调用频率 | 放置位置 | 特征 |
|------|----------|----------|------|
| Hot | ≥ 0.8 | GPU VRAM | 高频、需快速访问 |
| Warm | 0.3-0.8 | CPU DRAM | 中频、可容忍延迟 |
| Cold | < 0.3 | Disk/Remote | 低频、按需加载 |

### 2.3 kcake.cluster - 集群管理

**职责**：
- mDNS 节点发现
- 分片路由
- 权重流式传输

**核心类**：

```python
class ClusterManager:
    async def start()                           # 启动集群
    async def join_cluster(host, port)          # 加入集群
    async def register_shard(shard_id, layers)  # 注册分片
    async def request_shard(shard_id)           # 请求分片
    def get_cluster_status()                    # 集群状态
```

**集群拓扑发现流程**：
```
1. Master Node 启动，监听 8000 端口
2. Worker Node 启动，通过 mDNS 发现 master
3. Worker 连接到 master，注册自身信息
4. Master 维护节点列表和心跳
5. 推理请求到达 master，分发到 workers
```

### 2.4 kcake.devices - 设备抽象层

**职责**：
- 硬件能力探测
- 后端统一抽象
- 设备选择策略

**支持的设备后端**：

| 后端 | 优先级 | 适用场景 |

## 4. 关键算法

### 4.1 专家放置算法

```python
def schedule_expert(expert, available_devices):
    # 1. 根据频率分类
    expert_type = classify(expert.call_frequency)
    
    # 2. 选择目标设备类型
    target_type = {
        HOT: [CUDA, METAL],
        WARM: [CPU],
        COLD: [REMOTE, DISK]
    }[expert_type]
    
    # 3. 选择得分最高的可用设备
    candidates = filter_by_type(available_devices, target_type)
    return max(candidates, key=lambda d: d.score)
```

### 4.2 分片分发算法

```python
def distribute_shards(shards, nodes):
    # 1. 按节点计算能力排序
    sorted_nodes = sort_by_score(nodes)
    
    # 2. 计算每节点分片数
    shards_per_node = len(shards) / len(nodes)
    
    # 3. 轮询分配
    distribution = {}
    for i, shard in enumerate(shards):
        node_idx = min(i // shards_per_node, len(nodes) - 1)
        distribution[shard.id] = sorted_nodes[node_idx].id
    
    return distribution
```

kcake run <prompt> --model <model>

## 6. 错误处理

| 错误码 | 说明 | 处理策略 |
|--------|------|----------|
| E001 | 节点连接失败 | 重试3次，触发告警 |
| E002 | 分片获取超时 | 回退到其他节点 |
| E003 | 模型加载失败 | 返回错误，保留现场 |
| E004 | 推理超时 | 返回部分结果 |
| E005 | 设备内存不足 | 触发专家迁移 |
