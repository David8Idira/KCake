**版本**: V1.0
**日期**: 2026-04-02
**作者**: KCake Team

---

## 1. 项目背景

### 1.1 市场痛点

| 痛点 | 说明 | 现状 |
|------|------|------|
| GPU 资源昂贵 | 大模型推理需要高端 GPU | 企业成本高 |
| 设备闲置严重 | 手机/电脑等设备算力浪费 | 利用率 < 5% |
| 跨平台困难 | 不同硬件架构难以协同 | 碎片化严重 |
| 部署复杂 | 大模型部署门槛高 | 技术要求高 |

### 1.2 目标

KCake 旨在构建一个**异构设备集群化超大规模模型推理引擎**，将闲置的移动设备（手机、平板）、个人电脑、服务器等组成集群，协同运行超出单一设备能力范围的超大规模语言模型。

## 3. 非功能性需求

### 3.1 性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 单机吞吐 (8×L20) | ≥ 200 tok/s | DeepSeek-R1 FP8 |
| 集群聚合吞吐 | ≥ 50 tok/s | 1服务器+2手机 |
| 移动端延迟 | < 100ms/token | iPhone 15 Pro |
| 首 token 延迟 | < 1s | 冷启动 |
| 内存利用率 | ≥ 90% | GPU VRAM |

### 3.2 可用性

| 指标 | 目标值 |
|------|--------|
| 系统可用性 | ≥ 99.5% |
| 节点故障恢复时间 | < 30s |
| 集群自动发现时间 | < 10s |
| 权重同步时间 | < 5min/GB |

### 3.3 兼容性

- Python 版本: 3.10+
- PyTorch 版本: 2.0+
- Transformers: 4.35+
- 容器: Docker 20.10+

## 4. 用户场景

### 4.1 场景一：中小企业私有部署

**用户**：中小企业技术团队
**需求**：低成本运行大模型
**方案**：利用现有服务器 + 员工闲置电脑组建集群

### 4.2 场景二：开发者本地测试

**用户**：AI 开发者
**需求**：本地开发和测试大模型应用
**方案**：MacBook + 闲置 PC 组成小型集群

### 4.3 场景三：移动端边缘推理

**用户**：移动应用开发者
**需求**：在手机/平板上运行轻量模型
**方案**：iOS/Android 设备单独或组成集群

### 4.4 场景四：学术研究

**用户**：高校研究人员
**需求**：运行超大模型进行实验
**方案**：利用实验室多台服务器组建集群

## 5. 约束条件

| 约束类型 | 描述 |
|----------|------|
| 网络带宽 | 集群节点间建议 ≥ 100Mbps |
| 内存容量 | 单节点建议 ≥ 16GB RAM |
| 存储空间 | 权重缓存建议 ≥ 100GB SSD |
| 防火墙 | 需开放 mDNS (5353/UDP) |
