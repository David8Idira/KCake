## 2. 日常运维

### 2.1 启动服务

```bash
# 前台启动（测试用）
python -m kcake serve --model meta-llama/Llama-3.1-8B --cluster-key secret

# 后台启动（生产用）
nohup python -m kcake serve \
  --model meta-llama/Llama-3.1-8B \
  --cluster-key secret \
  --port 8000 \
  > /var/log/kcake.log 2>&1 &

# 检查进程
ps aux | grep kcake
```

### 2.2 停止服务

```bash
# 查找进程 PID
pid=$(pgrep -f "kcake serve")

# 优雅停止
kill -SIGTERM $pid

# 强制停止（如果优雅停止失败）
kill -SIGKILL $pid
```

### 2.3 重启服务

```bash
# 重启
systemctl restart kcake

# 或手动
pgrep -f "kcake serve" | xargs kill
sleep 2
python -m kcake serve --model meta-llama/Llama-3.1-8B --cluster-key secret &
```

### 2.4 日志管理

```bash
# 查看实时日志
tail -f /var/log/kcake.log

# 查看最近 100 行
tail -n 100 /var/log/kcake.log

# 日志轮转配置 (logrotate)
/var/log/kcake.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
}
```

## 3. 集群运维

### 3.1 添加节点

```bash
# 在新节点上执行
python -m kcake worker \
  --cluster-key your-secret \
  --name new-worker-node \
  --master-host master-ip \
  --master-port 8000
```

### 3.2 移除节点

```bash
# 方式一：在节点上停止 worker
# Ctrl+C 或 kill 进程

# 方式二：从 master 断开连接
# 节点会自动心跳超时后被标记为离线
```

### 3.3 节点故障处理

```bash
# 1. 检查节点状态
curl http://localhost:8000/cluster/status

# 2. 查看故障节点
# 状态为 "disconnected" 的节点

# 3. 尝试重新连接故障节点
python -m kcake worker \
  --cluster-key your-secret \
  --name failed-node \
  --master-host master-ip \
  --master-port 8000

# 4. 如果节点不可恢复，从集群移除
# 等待心跳超时自动移除
```

### 3.4 Master 节点故障切换

```bash
# 1. 停止原 master
# 在原 master 上执行
pgrep -f "kcake serve" | xargs kill

# 2. 选择一个 HYBRID 节点提升为 master
# 在新 master 上执行
python -m kcake serve \
  --model meta-llama/Llama-3.1-70B \
  --cluster-key your-secret \
  --name new-master \
  --port 8000

# 3. 通知所有 worker 连接新 master
# 在各 worker 上执行
python -m kcake worker \
  --cluster-key your-secret \
  --name worker-1 \
  --master-host new-master-ip \
  --master-port 8000
```

## 4. 性能调优

### 4.1 GPU 利用率优化

```bash
# 1. 启用 CUDA 优化
export CUDA_LAUNCH_BLOCKING=1

# 2. 设置 GPU 内存碎片整理
export PYTORCH_CUDA_ALLOC_CONF=max_split_size_mb:512

# 3. 启动时指定设备
python -m kcake serve \
  --model meta-llama/Llama-3.1-8B \
  --device-map cuda:0
```

### 4.2 内存优化

```bash
# 1. 启用量化减少内存占用
python -m kcake serve \
  --model meta-llama/Llama-3.1-70B \
  --quantization int4

# 2. 设置 KV Cache 大小
export KVCACHE_SIZE=8192

# 3. 启用前缀缓存
# 在配置中设置 enable_prefix_cache: true
```

### 4.3 吞吐量优化

```bash
# 1. 启用批量处理
export ENABLE_BATCH=true
export BATCH_SIZE=16

# 2. 启用并发推理
export MAX_CONCURRENT=4

# 3. 启动多 worker
python -m kcake serve \
  --model meta-llama/Llama-3.1-8B \
  --workers 4
```

## 5. 备份与恢复

### 5.1 备份

```bash
# 1. 备份模型缓存
rsync -avz ~/.cache/huggingface/ /backup/huggingface/

# 2. 备份配置
cp -r /opt/kcake/config /backup/kcake-config/

# 3. 备份日志
cp /var/log/kcake.log /backup/kcake-log-$(date +%Y%m%d)
```

### 5.2 恢复

```bash
# 1. 恢复模型缓存
rsync -avz /backup/huggingface/ ~/.cache/huggingface/

# 2. 恢复配置
cp -r /backup/kcake-config/* /opt/kcake/config/

# 3. 重启服务
systemctl restart kcake
```

## 6. 安全运维

### 6.1 防火墙配置

```bash
# 开放必要端口
sudo firewall-cmd --add-port=8000/tcp --permanent   # API
sudo firewall-cmd --add-port=5353/udp --permanent    # mDNS
sudo firewall-cmd --reload

# 限制 API 访问（仅内网）
sudo firewall-cmd --add-rich-rule='rule family="ipv4" source address="192.168.0.0/16" port port=8000 protocol=tcp accept'
```

### 6.2 集群密钥管理

```bash
# 使用强密钥（至少 32 字符）
CLUSTER_KEY=$(openssl rand -base64 32)

# 定期轮换密钥
# 1. 更新所有节点配置
# 2. 重启所有服务
```

### 6.3 API 认证

```bash
# 通过 Nginx 添加 Basic Auth
location / {
    auth_basic "KCake API";
    auth_basic_user_file /etc/nginx/.htpasswd;
    proxy_pass http://localhost:8000;
}
```

## 7. 故障排查

### 7.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| API 响应慢 | GPU 利用率低 | 检查批处理配置 |
| 节点不可见 | mDNS 未启动 | 安装 avahi-daemon |
| 模型加载失败 | 磁盘空间不足 | 清理磁盘或移动缓存 |
| GPU OOM | 显存不足 | 启用量化或减少并发 |
| 推理结果错误 | 模型损坏 | 重新下载模型 |

### 7.2 诊断命令

```bash
# 1. 系统状态
df -h                        # 磁盘空间
free -h                      # 内存
nvidia-smi                   # GPU 状态

# 2. 服务状态
curl http://localhost:8000/health
ps aux | grep kcake

# 3. 网络连接
netstat -tlnp | grep 8000   # 端口监听
ss -s                       # 连接统计

# 4. 日志分析
grep -i error /var/log/kcake.log
grep -i warning /var/log/kcake.log
```

### 7.3 性能分析

```bash
# 1. PyTorch Profiler
export KCAKE_PROFILE=true
python -m kcake serve ...

# 2. NVIDIA Profiler
nvprof --print-gpu-trace python -m kcake serve ...

# 3. 内存分析
python -c "import torch; torch.cuda.memory_summary()"
```

## 8. 升级维护

### 8.1 版本升级

```bash
# 1. 停止服务
systemctl stop kcake

# 2. 备份当前版本
cp -r /opt/kcake /opt/kcake-backup-$(date +%Y%m%d)

# 3. 更新代码
cd /opt/kcake
git pull origin main

# 4. 更新依赖
pip install -r requirements.txt
pip install -e .

# 5. 启动服务
systemctl start kcake
```

### 8.2 回滚

```bash
# 停止当前版本
systemctl stop kcake

# 恢复备份
rm -rf /opt/kcake
mv /opt/kcake-backup-YYYYMMDD /opt/kcake

# 重启服务
systemctl start kcake
```
