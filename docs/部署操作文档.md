## 1. 环境要求

### 1.1 硬件要求

| 节点类型 | CPU | 内存 | 存储 | GPU |
|----------|-----|------|------|-----|
| Master | 8核+ | 32GB+ | 100GB+ SSD | 建议 NVIDIA 8GB+ |
| Worker (GPU) | 4核+ | 16GB+ | 50GB+ | NVIDIA 6GB+ |
| Worker (CPU) | 4核+ | 8GB+ | 50GB+ | - |
| Worker (Edge) | 4核+ | 4GB+ | 10GB+ | - |

### 1.2 软件要求

- **操作系统**: Ubuntu 20.04+ / macOS 12+ / Windows 10+
- **Python**: 3.10, 3.11, 或 3.12
- **Docker**: 20.10+ (可选)
- **网络**: 集群节点间建议 ≥ 100Mbps

## 2. 单机部署

### 2.1 方式一：pip 安装

```bash
# 1. 克隆项目
git clone https://github.com/liq_idira/KCake.git
cd KCake

# 2. 创建虚拟环境
python3 -m venv kcake-env
source kcake-env/bin/activate  # Linux/macOS
# kcake-env\Scripts\activate   # Windows

# 3. 安装依赖
pip install --upgrade pip
pip install -r requirements.txt

# 4. 安装 KCake
pip install -e .

# 5. 验证安装
kcake --version
```

### 2.2 方式二：Docker 部署

```bash
# 1. 构建镜像
docker build -t kcake:latest .

# 2. 运行容器
docker run -d \
  --name kcake \
  -p 8000:8000 \
  -v ~/.cache/huggingface:/root/.cache/huggingface \
  kcake:latest \
  serve --model meta-llama/Llama-3.1-8B --cluster-key mysecret
```

## 3. 集群部署

### 3.1 方式一：Docker Compose

```bash
# 1. 克隆项目
git clone https://github.com/liq_idira/KCake.git
cd KCake

# 2. 创建 .env 文件
echo "CLUSTER_KEY=mysecret123" > .env

# 3. 启动集群
docker-compose up -d

# 4. 查看状态
docker-compose ps
```

### 3.2 方式二：手动部署

#### 3.2.1 部署 Master 节点

```bash
# 1. 在主节点服务器上执行
cd /opt/kcake

# 2. 启动主节点
python -m kcake serve \
  --model meta-llama/Llama-3.1-70B \
  --cluster-key mysecret123 \
  --host 0.0.0.0 \
  --port 8000 \
  --name master-node
```

#### 3.2.2 部署 Worker 节点

```bash
# 1. 在 worker 节点上执行
cd /opt/kcake

# 2. 启动 worker 节点
python -m kcake worker \
  --cluster-key mysecret123 \
  --name worker-gpu-1 \
  --master-host 192.168.1.100 \
  --master-port 8000
```

#### 3.2.3 移动端部署 (iOS/Android)

```bash
# iOS (使用 Xcode 构建)
# 1. 下载 KCake iOS 源码
# 2. 使用 Xcode 打开 kcake-ios/KCake.xcodeproj
# 3. 配置签名和证书
# 4. 构建并运行

# Android
# 1. 下载 KCake Android 源码
# 2. 导入 Android Studio
# 3. Sync Gradle
# 4. Build APK
```

## 4. 配置说明

### 4.1 环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `CLUSTER_KEY` | 集群认证密钥 | - |
| `KCACHE_HOME` | 模型缓存目录 | `~/.cache/huggingface` |
| `LOG_LEVEL` | 日志级别 | `INFO` |
| `MAX_WORKERS` | 最大工作线程数 | CPU 核心数 |

### 4.2 命令行参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--model` | 模型名称或路径 | `meta-llama/Llama-3.1-8B` |
| `--cluster-key` | 集群密钥 | `mysecret123` |
| `--host` | 监听地址 | `0.0.0.0` |
| `--port` | 监听端口 | `8000` |
| `--device-map` | 设备映射 | `auto` / `cuda` / `cpu` |
| `--dtype` | 数据类型 | `float16` / `bfloat16` |
| `--quantization` | 量化类型 | `int4` / `int8` / `fp8` |

## 6. 常见问题

### 6.1 模型下载失败

```bash
# 设置代理
export HTTP_PROXY=http://proxy.example.com:8080
export HTTPS_PROXY=http://proxy.example.com:8080

# 或手动下载
huggingface-cli download meta-llama/Llama-3.1-8B
```

### 6.2 端口被占用

```bash
# 查看端口占用
lsof -i :8000

# 使用其他端口
python -m kcake serve --port 8001
```

### 6.3 GPU 未识别

```bash
# 检查 NVIDIA 驱动
nvidia-smi

# 检查 PyTorch CUDA
python -c "import torch; print(torch.cuda.is_available())"
```

### 6.4 mDNS 发现失败

```bash
# 安装 avahi-daemon (Linux)
sudo apt install avahi-daemon

# 开放防火墙
sudo firewall-cmd --add-port=5353/udp
```

docker stop kcake

docker rmi kcake

# 清理缓存
rm -rf ~/.cache/kcake
